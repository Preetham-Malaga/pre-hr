# CorpHRMS Project Rules

## Employee ID Logic
- Employee ID must be sequential.
- Format: EMP-001, EMP-002, EMP-003...
- Generated by Supabase trigger/function.
- Never use Date.now() for employee IDs.

## Employee Add
- AddEmployee.tsx creates employees.
- Employee ID shown before save.
- Employee ID generated by database.

## Employee Edit
- EditEmployee.tsx uses separate page.
- No popup editing.
- Update existing employee.

## Employee View
- Same layout as Add Employee.
- Show all employee fields.
- Show uploaded documents.

## Documents
- Stored in Supabase Storage.
- View and Download must work.

## Important
- Don't break Employee ID logic.
- Don't convert Edit back to popup.
- Don't remove Supabase trigger.
